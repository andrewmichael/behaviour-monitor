class DriftDetector:
    def to_dict(self) -> dict[str, Any]:
        return {
            "sensitivity": self._sensitivity,
            "states": {entity_id: state.to_dict() for entity_id, state in self._states.items()},
        }

    @classmethod
    def from_dict(cls, data: dict[str, Any]) -> "DriftDetector":
        detector = cls(sensitivity=data.get("sensitivity", SENSITIVITY_MEDIUM))
        for entity_id, state_data in data.get("states", {}).items():
            detector._states[entity_id] = CUSUMState.from_dict(state_data)
        return detector

<feature>
  <name>CorrelationDetector with PMI-based discovery</name>
  <files>custom_components/behaviour_monitor/correlation_detector.py, tests/test_correlation_detector.py</files>
  <behavior>
    Record events and co-occurrence detection:
    - record_event("sensor.a", t1, {"sensor.a": t1, "sensor.b": t0}) where t1-t0 < window => co-occurrence count for (sensor.a, sensor.b) increments
    - record_event("sensor.a", t1, {"sensor.a": t1, "sensor.b": t0}) where t1-t0 > window => no co-occurrence counted, solo count increments
    - record_event("sensor.a", t1, {}) with no other entities seen => no pairs created (lazy discovery)

    PMI recomputation:
    - recompute() with pair having co_occurrences=47, total_events=62 and appropriate base rates => PMI > 1.0, pair is "learned"
    - recompute() with pair having co_occurrences=3 (below MIN_CO_OCCURRENCES=10) => pair NOT promoted to learned
    - recompute() with pair having low PMI (< PMI_THRESHOLD=1.0) despite sufficient count => pair NOT learned

    Sensor attribute output:
    - get_correlation_groups() with learned pairs => list of dicts with "entities", "co_occurrence_rate", "total_observations"
    - get_correlation_groups() with no learned pairs => empty list
    - get_correlated_entities("sensor.a") when sensor.a has a learned partner => list containing partner entity IDs

    Persistence:
    - to_dict() => dict with "co_occurrence_window_seconds", "pairs" keys; pairs keyed by sorted "|"-joined entity IDs
    - from_dict(to_dict()) round-trips without data loss (co_occurrences, solo_counts, first_observed preserved)
    - from_dict({}) => empty detector (graceful handling of missing keys)

    Edge cases:
    - Sorted tuple keys: record_event for (A,B) and (B,A) updates same CorrelationPair
    - Zero HA imports: file contains no "homeassistant" import
  </behavior>
  <implementation>
    Create correlation_detector.py following the established detector pattern:

    1. CorrelationPair dataclass with fields:
       - entities: tuple[str, str] (sorted pair)
       - co_occurrences: int (times both fired within window)
       - solo_counts: dict[str, int] (per-entity solo fire counts)
       - first_observed: str | None (ISO timestamp)
       - Properties: total_events, co_occurrence_rate
       - to_dict() / from_dict() serialization

    2. CorrelationDetector class:
       - __init__(co_occurrence_window_seconds, min_observations, pmi_threshold)
       - Constructor defaults from const.py: DEFAULT_CORRELATION_WINDOW, MIN_CO_OCCURRENCES, PMI_THRESHOLD
       - _pairs: dict[tuple[str, str], CorrelationPair] — lazy, keyed by sorted entity tuple
       - _learned_pairs: set[tuple[str, str]] — pairs that passed PMI+count gate

       - record_event(entity_id: str, timestamp: datetime, all_last_seen: dict[str, datetime]) -> None
         For each other entity in all_last_seen where abs(timestamp - last_seen) <= window:
           Get or create CorrelationPair for sorted(entity_id, other_id)
           Increment co_occurrences
         For the firing entity, if no co-occurrence with a known partner: increment solo_counts

       - recompute() -> None
         For each pair with co_occurrences >= min_observations:
           Compute PMI = log2(P(co) / (P(a) * P(b)))
           Where P(co) = co_occurrences / total_events, P(a) and P(b) = (co + solo) / total
           If PMI >= pmi_threshold: add to _learned_pairs
           Else: remove from _learned_pairs

       - get_correlation_groups() -> list[dict[str, Any]]
         Return list of {"entities": [a, b], "co_occurrence_rate": float, "total_observations": int}
         for all learned pairs

       - get_correlated_entities(entity_id: str) -> list[str]
         Return entity IDs that are learned partners of the given entity

       - to_dict() / from_dict() following DriftDetector pattern
         Pair keys: sorted entity IDs joined by "|"
  </implementation>
</feature>
