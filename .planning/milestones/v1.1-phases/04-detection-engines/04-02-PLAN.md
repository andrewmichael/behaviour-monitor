Then create `drift_detector.py` (GREEN phase). Implementation follows research Pattern 3:
- `CUSUMState` dataclass with `s_pos`, `s_neg`, `days_above_threshold`, `last_update_date`, `reset()`, `to_dict()`, `from_dict()`
- `DriftDetector.__init__` reads `(k, h)` from const.py `CUSUM_PARAMS[sensitivity]`, falls back to "medium" if invalid
- `_states: dict[str, CUSUMState]` for per-entity accumulator state
- `get_or_create_state(entity_id)` returns existing or new CUSUMState
- `reset_entity(entity_id)` calls `state.reset()` on the entity's CUSUMState
- `check(entity_id, routine, today, now)` -> AlertResult | None:
  1. Skip if `state.last_update_date == today.isoformat()` (idempotent)
  2. Compute baseline rates from routine event_times (all unique dates excluding today)
  3. Guard: return None if fewer than MIN_EVIDENCE_DAYS baseline dates
  4. Guard: return None if baseline_mean == 0
  5. Compute z = (today_rate - baseline_mean) / baseline_stdev
  6. Update S_pos = max(0, S_pos + z - k), S_neg = max(0, S_neg - z - k)
  7. If S_pos > h or S_neg > h: increment days_above_threshold, else reset to 0
  8. If days_above_threshold >= MIN_EVIDENCE_DAYS: produce AlertResult
- `_compute_baseline_rates(routine, exclude_today)` scans all slots' event_times, groups by date, counts per day
- `_drift_severity(days)` static: 3-6 -> MEDIUM, >=7 -> HIGH, <3 -> LOW
- `to_dict()` / `from_dict()` for persistence

Import from const.py: `from .const import CUSUM_PARAMS, MIN_EVIDENCE_DAYS, SENSITIVITY_MEDIUM`
Use `statistics.stdev` for baseline standard deviation (already in stdlib, used by routine_model).
