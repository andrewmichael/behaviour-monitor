Delete the entire 1,213-line coordinator.py and write a new one from scratch. Target ~300 lines. Structure:

**Imports (~15 lines):**
Import from HA: DataUpdateCoordinator, Store, EVENT_STATE_CHANGED, dt as dt_util.
Import from local: RoutineModel, AcuteDetector, DriftDetector, AlertResult, AlertSeverity, AlertType, CUSUMState, is_binary_state, and all needed CONF_*/DEFAULT_* from const.py.

**Class BehaviourMonitorCoordinator(DataUpdateCoordinator) (~285 lines):**

`__init__(self, hass, entry)` (~30 lines):
- Call super().__init__ with name=DOMAIN, update_interval=timedelta(seconds=UPDATE_INTERVAL).
- Read config from entry.data: monitored_entities, inactivity_multiplier, drift_sensitivity, history_window_days, enable_notifications, notify_services, notification_cooldown, min_notification_severity.
- Instantiate: self._routine_model = RoutineModel(history_window_days), self._acute_detector = AcuteDetector(inactivity_multiplier), self._drift_detector = DriftDetector(drift_sensitivity).
- Initialize: self._last_seen: dict[str, datetime] = {}, self._notification_cooldowns: dict[tuple[str, str], datetime] = {}, self._holiday_mode = False, self._snooze_until: datetime | None = None, self._today_count = 0, self._today_date = None, self._last_notification_info = {"timestamp": None, "type": None}, self._welfare_debounce: dict[str, int] = {}, self._current_welfare_status = "ok", self._store = Store(hass, STORAGE_VERSION, f"{STORAGE_KEY}.{entry.entry_id}"), self._unsub_state_changed = None.
- Expose self.monitored_entities as a property returning the configured list.
- Add analyzer shim for sensor.py backward compat: create a simple object with `is_learning_complete()` method that returns `self._routine_model.learning_status() == "ready"`. Use a namedtuple or simple class: `self.analyzer = type('_Shim', (), {'is_learning_complete': lambda s: self._routine_model.learning_status() == "ready"})()`

`async_setup(self)` (~35 lines):
- Load stored data via self._store.async_load().
- If stored data exists: restore self._routine_model from stored_data["routine_model"] using RoutineModel.from_dict(). Restore cusum_states from stored_data.get("cusum_states", {}) into self._drift_detector._states via CUSUMState.from_dict(). Restore coordinator state (holiday_mode, snooze_until, last_seen, last_notification_info, notification_cooldowns) from stored_data.get("coordinator", {}).
- If no stored data and routine model is empty: bootstrap from recorder using the existing pattern from the old coordinator — call recorder get_instance, get_significant_states for history_window_days, feed states into routine_model.record(). Use try/except around recorder access; log warning if unavailable.
- Subscribe to state changes: self._unsub_state_changed = self.hass.bus.async_listen(EVENT_STATE_CHANGED, self._handle_state_changed).

`async_shutdown(self)` (~10 lines):
- Unsubscribe state listener. Call self._save_data().

`async _save_data(self)` (~20 lines):
- Build storage dict with keys: "routine_model" (self._routine_model.to_dict()), "cusum_states" ({eid: s.to_dict() for eid, s in self._drift_detector._states.items()}), "coordinator" (holiday_mode, snooze_until as ISO or None, last_seen as {eid: dt.isoformat()}, last_notification_info, notification_cooldowns as {f"{k[0]}|{k[1]}": v.isoformat()}).
- await self._store.async_save(data).

`_handle_state_changed(self, event)` (~20 lines):
- Extract entity_id from event.data. If not in monitored_entities, return.
- Record to routine_model: self._routine_model.record(entity_id, new_state, dt_util.now(), is_binary_state(entity_id)).
- Update self._last_seen[entity_id] = dt_util.now().
- Increment self._today_count (after checking midnight reset).
- Schedule refresh: self.hass.async_create_task(self.async_request_refresh()).

`async _async_update_data(self)` (~40 lines):
- Reset daily count if date changed.
- If holiday mode or snoozed: return self._build_safe_defaults().
- Try: alerts = self._run_detection(now). Handle alerts. Return self._build_sensor_data(alerts, now).
- Except: log error, return self._build_safe_defaults().

`_run_detection(self, now)` (~20 lines):
- Loop over monitored_entities. Get routine via self._routine_model._entities.get(entity_id) (coordinator is the only consumer; routine may be None for new entities). If None, skip.
- Call self._acute_detector.check_inactivity(entity_id, routine, now, self._last_seen.get(entity_id)), check_unusual_time, self._drift_detector.check(entity_id, routine, now.date(), now). Collect non-None results.

`async _handle_alerts(self, alerts, now)` (~30 lines):
- If not self._enable_notifications: return.
- Filter alerts through severity gate (_meets_severity_gate) and cooldown gate (_should_notify).
- Apply welfare debounce: derive new welfare status from alerts, compare to current, increment/reset debounce counter per status, only fire welfare notification after WELFARE_DEBOUNCE_CYCLES.
- For notifiable alerts: send notification, update cooldown, update last_notification_info.

`_should_notify(self, alert)` (~8 lines):
- Key = (alert.entity_id, alert.alert_type.value).
- Check cooldown: if last send time exists and elapsed < cooldown minutes, return False. Else True.

`_meets_severity_gate(self, alert)` (~8 lines):
- Map min_notification_severity string to AlertSeverity using: {"minor": AlertSeverity.LOW, "moderate": AlertSeverity.LOW, "significant": AlertSeverity.MEDIUM, "critical": AlertSeverity.HIGH}. Default to MEDIUM.
- Compare alert.severity order to gate.

`async _send_notification(self, alerts)` (~15 lines):
- Title = f"Behaviour Monitor: {len(alerts)} alert(s)".
- Lines = [f"- [{a.severity.value.upper()}] {a.explanation}" for a in alerts].
- Send via persistent_notification.create.
- For each notify_service: split domain.service, call hass.services.async_call.

`_derive_welfare(self, alerts)` (~15 lines):
- If no alerts: return {"status": "ok", "reasons": [], "summary": "No active alerts", "recommendation": "", "entity_count_by_status": {}}.
- Determine max severity from alerts. Map: LOW->"check_recommended", MEDIUM->"concern", HIGH->"alert".
- Return dict with status, reasons (alert explanations), summary, recommendation.

`_build_sensor_data(self, alerts, now)` (~40 lines):
- Build and return dict with ALL keys listed in the sensor data contract above.
- last_activity: max of self._last_seen values as ISO string, or None.
- activity_score: routine_model.overall_confidence(now) * 100.
- anomaly_detected: len(alerts) > 0.
- anomalies: [a.to_dict() for a in alerts].
- confidence: routine_model.overall_confidence(now) * 100.
- daily_count: self._today_count.
- welfare: self._derive_welfare(alerts).
- routine: compute progress_percent, expected_by_now, actual_today, expected_full_day from routine_model data. actual_today = self._today_count. expected_by_now and expected_full_day: sum across entities of daily_activity_rate for current weekday, scaled by hour proportion. If no data, use 0s. status: "on_track" if actual >= expected_by_now * 0.7 else "below_expected". summary: f"{actual_today} of ~{expected_full_day} expected activities".
- activity_context: compute from self._last_seen. time_since = now - max(last_seen). typical_interval from routine. Format nicely.
- entity_status: list of per-entity dicts with entity_id, status, last_seen.
- stat_training: derive from routine_model. first_observation from oldest entity.first_observation. days_elapsed, total_days = history_window_days, days_remaining, complete, formatted.
- ml_status: {"enabled": False}.
- cross_sensor_patterns: [].
- last_notification: self._last_notification_info.
- holiday_mode, snooze_active, snooze_until.
- ml_status_stub, ml_training_stub, cross_sensor_stub: static values.
- learning_status, baseline_confidence.

`_build_safe_defaults(self)` (~20 lines):
- Return dict with safe values for all keys (see Research Pattern safe defaults dict).

`is_snoozed(self)` (~3 lines): Return bool.

Holiday/snooze methods (~20 lines total):
- async_enable_holiday_mode, async_disable_holiday_mode, async_snooze, async_clear_snooze — same simple pattern as old coordinator. Set flag/time, save data, fire event, request refresh.

`async_routine_reset(self, entity_id)` (~5 lines):
- self._drift_detector.reset_entity(entity_id). Log warning. Fire event. Save data.

TOTAL: ~300 lines.

- Never return None from _async_update_data — always return a dict.
- Always wrap _async_update_data body in try/except returning _build_safe_defaults on error.
- Initialize _last_seen as empty dict in __init__, populate from storage in async_setup.
- Track _today_count and _today_date for midnight reset.
