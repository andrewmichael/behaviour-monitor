## Tech Debt

### Phase 5: Integration

1. **Post-bootstrap persist missing** (ROUTINE-02)
   - `coordinator.py` `async_setup` does not call `_save_data()` after `_bootstrap_from_recorder()`
   - Risk: if HA restarts before any monitored entity changes state, bootstrap re-runs and doubles all historical observations
   - Fix: add `await self._save_data()` after `_bootstrap_from_recorder()` returns

2. **Legacy constants dead code** (INFRA-01)
   - `const.py` lines 129-184 contain `SENSITIVITY_THRESHOLDS`, `ML_CONTAMINATION`, `MAX_VARIANCE_MULTIPLIER`, and other constants with zero consumers
   - The `analyzer.py` and `ml_analyzer.py` files that imported these are deleted
   - Fix: delete the legacy constants block

3. **Coordinator stub keys unused** (INFRA-02)
   - `_build_sensor_data()` emits `ml_status_stub`, `ml_training_stub`, `cross_sensor_stub` keys
   - `sensor.py` deprecated sensor lambdas use hardcoded literals instead of reading these keys
   - Fix: remove the coordinator stub keys (sensor hardcoded approach is simpler)

**Total: 3 items, 1 phase**
