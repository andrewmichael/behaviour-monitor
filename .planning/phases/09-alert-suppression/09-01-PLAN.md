<objective>
Add fire-once-then-throttle suppression logic to the coordinator. An alert key (entity_id + alert_type) fires once, then is suppressed for the repeat interval while the condition remains active. When the condition clears (key absent from current detection results), the suppression entry is removed so the next occurrence fires immediately.

Purpose: Eliminates notification fatigue where the same active alert fires every 60-second polling cycle.
Output: Updated const.py (new constants), updated coordinator.py (suppression logic), updated tests.
</objective>
